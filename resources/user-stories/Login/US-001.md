# User Story: Admin Login

## Application Details

- **Application Name:** 360 Auto Marketplace - Admin Panel
- **Environment:** QA
- **Admin URL:** https://marketplace-admin.appworkdemo.com/

### Login Credentials

| Field | Value |
|--------|-------|
| Email | admin@360automarketplace.com |
| Password | Admin@123 |

---

# User Story Information

## Story ID
US-ADMIN-001

## Story Title
Admin Login

## Module
Authentication

## Priority
High

## Story Description

**As an Admin,**
I want to log in using my registered email address and password,
**So that**
I can securely access the Admin Dashboard and manage the marketplace.

---

# Business Rules

1. Only registered admin accounts can log in.
2. Email must be a valid registered admin email.
3. Password is case-sensitive.
4. Both Email and Password are mandatory.
5. Successful login redirects the admin to the Dashboard.
6. Invalid credentials should not allow login.
7. Forgot Password should be available on the Login page.

---

# Functional Requirements

## Login Screen

The login page should contain:

- Email Address field
- Password field
- Login button
- Forgot Password link

---

## Login Process

### Step 1

Admin opens the Admin Login URL.

### Step 2

Admin enters:

- Registered Email
- Password

### Step 3

Admin clicks the **Login** button.

### Step 4

System validates credentials.

### Step 5

If credentials are correct

System should:

- Authenticate the user
- Create user session
- Redirect to Dashboard

---

## Failed Login

If Email or Password is incorrect,

System should

- Prevent login
- Stay on Login page
- Display proper error message

Example:

> Invalid email or password.

---

## Forgot Password

When Admin clicks **Forgot Password**

System should

- Redirect user to Forgot Password page

---

# Acceptance Criteria

## AC-001

Given Admin is on Login page

When valid Email and Password are entered

Then Login should be successful.

---

## AC-002

Given Admin enters invalid Email

When Login button is clicked

Then login should fail.

---

## AC-003

Given Admin enters invalid Password

When Login button is clicked

Then login should fail.

---

## AC-004

Given both Email and Password are invalid

When Login button is clicked

Then login should fail.

---

## AC-005

Given Email field is blank

When Login button is clicked

Then validation message should appear.

---

## AC-006

Given Password field is blank

When Login button is clicked

Then validation message should appear.

---

## AC-007

Given both fields are blank

When Login button is clicked

Then required field validation should appear.

---

## AC-008

Given Admin clicks Forgot Password

Then Forgot Password page should open.

---

# Validation Rules

## Email Field

- Mandatory
- Accept valid email format only
- Registered admin email only

Example

admin@360automarketplace.com

---

## Password Field

- Mandatory
- Case-sensitive
- Hidden by default

# Positive Test Scenarios

- Login with valid credentials
- Login using registered admin email
- Dashboard opens successfully
- Forgot Password navigation works

---

# Negative Test Scenarios

- Invalid email
- Invalid password
- Both invalid
- Empty email
- Empty password
- Empty fields
- Invalid email format
- SQL Injection attempt
- XSS input
- Extra spaces in email
- Password with wrong case

---

# Edge Test Scenarios

- Very long email
- Very long password
- Copy/Paste credentials
- Browser refresh
- Multiple login attempts
- Session timeout
- Browser Back button after logout

---

# Security Test Scenarios

- Password should be masked
- HTTPS enabled
- Session created after login
- Session destroyed after logout
- Unauthorized dashboard access should be blocked
- Direct Dashboard URL without login should redirect to Login page
